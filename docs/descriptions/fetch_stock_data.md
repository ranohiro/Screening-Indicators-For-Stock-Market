# 銘柄データ一括抽出・差分分析スクリプト（`fetch_stock_data.py`）仕様・解説書

このドキュメントは、データベースから特定の銘柄に関する日足・財務・信用残高などの時系列データおよび属性情報を一括で抽出し、期間差分（騰落差分）を算出して1つのCSVファイルとして出力するユーティリティスクリプト [fetch_stock_data.py](file:///Users/hiranotakahiro/Projects/銘柄スクリーニング検証/src/fetch_stock_data.py) の技術仕様および使用方法をまとめたものです。

---

## 1. 目的と主な機能
本スクリプトは、特定の銘柄コードを指定して、`stock_data.db` 内に分散して保存されている様々な時系列データをマージし、データ分析や機械学習のインプット、あるいはExcel等での可視化に適したワイドフォーマット（横持ち）のCSVファイルを出力します。

主な機能：
- **複数時系列テーブルの統合**: 8種類の時系列テーブルから対象銘柄のデータを抽出。
- **日付キーの統一とアウタージョイン**: 各テーブルで異なる日付カラム名を `date` に統一し、外部結合（Outer Join）することで、休日や開示日の有無に関わらず全データを日付順に統合。
- **期間差分（モメンタム）の自動計算**: 数値カラムに対し、1日（1営業日）、1週（5営業日）、1ヶ月（20営業日）の騰落差分（差分値）を自動的に追加。
- **列名の衝突防止**: 列名にテーブル名プレフィックスを自動付与。
- **マスター情報の結合**: 銘柄基本情報（`companies` テーブル）を結合。

---

## 2. 処理詳細・アルゴリズム

### 2.1. 抽出対象テーブルと日付キー
スクリプトは以下のテーブルを巡回し、指定された銘柄コードに一致するレコードを抽出します。

| 抽出対象テーブル名 | 日付キー列名（抽出時） | 処理内容 |
| :--- | :--- | :--- |
| `daily_prices` | `date` | 日足（始値、高値、安値、終値、出来高、時価総額等）の抽出 |
| `daily_financials` | `date` | PER、PBR、EPS、配当利回り等の抽出 |
| `weekly_margin` | `date` | 信用買い残・売り残、信用倍率等の抽出 |
| `financial_results` | `date` | 決算の業績実績および予想数値の抽出 |
| `stock_daily_indicators` | `date` | 騰落率情報や時価総額分類の抽出 |
| `daily_trade_indicators` | `date` | テクニカル指標（RSI, MACD, 移動平均乖離率等）の抽出 |
| `stock_eps_revisions` | `revision_date` | アナリストや会社予想によるEPS修正履歴を抽出し、キー名を `date` に変換 |
| `corporate_disclosures` | `disclosure_date` | 適時開示データ（AIスコア含む）を抽出し、キー名を `date` に変換。※要約やPDF全文などの巨大なテキスト列（`ai_summary`, `title`, `document_url`, `created_at`, `pdf_text`）はCSV肥大化防止のため除外 |

---

### 2.2. 期間差分の算出
抽出された各テーブルの DataFrame 内の数値型列（`id`, `code`, `date` 等のID列を除く）に対して、以下の差分列が自動的に算出・追加されます。

- **`{カラム名}_1d_diff`**: 前日比（1行前の値との差分）
- **`{カラム名}_1w_diff`**: 前週比（5行前の値との差分）
- **`{カラム名}_1m_diff`**: 前月比（20行前の値との差分）

---

### 2.3. マージおよびプレフィックス付与
- 統合時のカラム名衝突を避けるため、`code` と `date` 以外のすべてのカラムに対して、`{テーブル名}_{元のカラム名}` の形式でプレフィックスが付与されます。
  - 例: `daily_prices` テーブルの `close` 列 $\rightarrow$ `daily_prices_close` / `daily_prices_close_1d_diff`
- すべての時系列テーブルを `code` と `date` をキーにして `outer join` でマージします。
- 最後に、`companies` テーブルから上場市場や業種等のマスタ情報を結合し（列名プレフィックス: `companies_`）、日付順にソートします。

## 3. 使用方法（コマンドライン）

ターミナルから引数を指定して実行します。

### 基本コマンド
```bash
python src/fetch_stock_data.py <銘柄コード>
```

### オプション
- `--db`: 読み込み先 SQLite データベースファイルのパスを指定（デフォルト: `../data/stock_data.db`）。
- `--out`: 出力先 CSV ファイルのパスを指定（デフォルト: `stock_{銘柄コード}_all_data.csv`）。

### 実行例
```bash
# 村田製作所(6981)のデータを抽出し、カスタムパスのCSVに保存する
python src/fetch_stock_data.py 6981 --db data/stock_data.db --out output/murata_all.csv
```

---

## 4. 出力ファイル（CSV）のエンコーディング
出力されるCSVは **`utf-8-sig` (BOM付きUTF-8)** エンコーディングで保存されます。これにより、日本語が含まれていても Excel 等で文字化けすることなく直接開くことができます。
