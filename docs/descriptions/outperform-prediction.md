# 大相場銘柄の予測と特徴量重要度の検証ノートブック（`outperform_prediction.ipynb`）仕様・解説書

このドキュメントは、特定銘柄のテーマに関連する銘柄リストを取得し、大相場銘柄（セクターやテーマ内平均をアウトパフォームする銘柄）の予測モデルを作成・検証するためのノートブック [outperform_prediction.ipynb](file:///Users/hiranotakahiro/Projects/銘柄スクリーニング検証/notebooks/outperform_prediction.ipynb) の技術仕様、分析パイプライン、および出力結果の解釈方法をまとめたものです。

---

## 1. 目的と位置づけ
本ノートブックの主な目的は以下の通りです。
- **大相場銘柄の特徴量検証**: 出来高の急増率や移動平均乖離率、ボラティリティ、信用取引の需給、財務指標（PBR, PER）などが、大相場を形成する銘柄の先行指標になり得るかを検証する。
- **予測モデルの構築**: 機械学習モデル（L1正則化ロジスティック回帰およびランダムフォレスト）を用いて、対象テーマ群の中で将来アウトパフォームする銘柄を予測する。
- **特徴量重要度の可視化**: どの説明変数が予測において重要であるかを分析し、スクリーニング条件に組み込むべきパラメータのヒントを得る。

---

## 2. 前提データ構造と設定パラメータ

### 2.1 データソース
計算には以下の2つのデータソースを使用します。
1. **テーマ・タグマスター (`ticker_dictionary.json`)**
   - 各銘柄コードのテーマタグ（`layered_tags` の `Sector_Major` および `End_Market`、または `tags`）が定義されたマスター辞書。
2. **日足株価データベース (`stock_data.db` 内の各テーブル)**
   - `daily_prices`: 日付 `date`, 銘柄コード `code`, 終値 `close`, 出来高 `volume` を保持。
   - `daily_trade_indicators`: 出来高変化率 `vol_inc_pct` やATR正規化値 `atr_norm`、信用倍率 `margin_ratio` を保持。
   - `daily_financials`: 財務情報（PBR実績 `pbr_actual`, PER予想 `per_forecast`）を保持。

### 2.2 設定定数（初期設定）
| 定数名 | デフォルト値 | 説明 |
| :--- | :--- | :--- |
| `TARGET_CODE` | `"1301"` | 分析の起点となるターゲット銘柄コード。 |
| `START_DATE` | `"20230101"` | 特徴量を評価する対象期間の開始日。 |
| `END_DATE` | `"20231231"` | 特徴量を評価する対象期間の終了日。 |
| `FORWARD_DAYS` | `20` | 何日後のリターンでアウトパフォームを判定するか（予測期間）。 |
| `OUTPERFORM_THRESHOLD` | `1.2` | テーマ内平均リターンの何倍を「大相場（正例）」と定義するか。 |

## 3. 分析パイプラインの詳細

### 【Step 1: 同テーマ銘柄の抽出（積集合ロジック）】
起点となる `TARGET_CODE`（例: 極洋 `1301`）の持つすべてのタグ（`target_tags`）を取得し、それら**すべてのタグを含んでいる銘柄**を抽出します。
- **マッチング条件**: `target_tags.issubset(item_tags)` (積集合条件)
- **効果**: 関連性の低い銘柄を徹底的に排除し、ターゲット銘柄と非常に近い「純粋な同テーマ銘柄群」に絞り込みます。

### 【Step 2: 時系列データ取得とインジケータ計算】
指定された `START_DATE` より前（デフォルトで60日分の `buffer_days`）のデータも含めてデータベースからロードします。
- **目的**: 20日移動平均などの指標（ボラティリティや出来高比率）を、指定期間の開始日（`START_DATE`）時点で正しく計算できるようにするため。
- **計算する特徴量**:
  - `vol_inc_pct`: 当日出来高変化率
  - `atr_norm`: ATR（アベレージ・トゥルー・レンジ）の正規化値
  - `margin_ratio`: 信用倍率
  - `pbr` / `per`: 財務指標
  - `volume_ratio_20d`: 20日平均出来高に対する当日出来高の割合（当日出来高 / 20日平均出来高）
  - `close_dev_20d`: 20日移動平均に対する終値乖離率（(当日終値 - 20日平均) / 20日平均）

### 【Step 3: 正解ラベル（目的変数）の作成】
1. 各取引日 $T$ において、各銘柄の $T$ 日終値から $T + \text{FORWARD\_DAYS}$ 日（20日後）の終値へのリターンを計算します。
2. 同一取引日（`date`）における「テーマ内銘柄の平均リターン」を算出します。
3. 以下の条件を満たす銘柄を「大相場銘柄（正例: `1`）」とし、満たさないものを「通常銘柄（負例: `0`）」とラベリングします。
   $$\text{正例条件} = (\text{将来リターン} > \text{テーマ内平均リターン} \times \text{OUTPERFORM\_THRESHOLD}) \ \text{かつ} \ (\text{将来リターン} > 0)$$

### 【Step 4: 特徴量エンジニアリング（Z-score正規化・次元削減）】
- **日付ごとのZ-score正規化**:
  相場全体の地合い（例: 全体相場が急落した日にPBRやボラティリティの絶対値が歪むなど）の影響を排除するため、**取引日（`date`）ごとに同一日付内の銘柄間で Z-score 正規化**を適用します。
- **主成分分析 (PCA)**:
  多重共線性（特徴量同士の強い相関）の緩和とデータの可視化を目的に、正規化した特徴量を2次元に圧縮（`pca1`, `pca2`）します。

### 【Step 5: モデルの時系列割分割と評価】
時系列のリーケージ（未来のデータで過去を予測してしまうこと）を防ぐため、データをシャッフルせずに日付順で分割します。
- **訓練データ**: 全体期間の前半 80% の取引日
- **テストデータ**: 全体期間の後半 20% の取引日
- **使用モデル**:
  1. **L1正則化ロジスティック回帰 (Lasso)**: 係数をスパース（不要な特徴量の係数をゼロ）にしながら線形な関係を捉える。
  2. **ランダムフォレスト**: 特徴量間の非線形な関係や交互作用を捉える。

## 4. 可視化結果（出力）の解釈ガイド

ノートブックの実行によって得られる3つのグラフは、以下のように解釈します。

### ① PCA Projection of Normalized Features (PCA投影散布図)
- **グラフの概要**:
  多次元の特徴量（ボラティリティ、出来高比率、財務指標など）を2次元の座標に圧縮した散布図です。
  - 青い点：通常銘柄（負例）
  - 赤い点：大相場銘柄（正例）
- **解釈 of ポイント**:
  - **赤と青の領域が綺麗に分かれている場合**:
    特徴量選択とZ-score正規化が非常にうまく機能しており、単純な境界線によって大相場銘柄を高い確率で分類できる可能性を示しています。
  - **赤と青が完全に混ざり合っている場合**:
    現在定義している特徴量（テクニカル/ファンダメンタルズ）だけでは、大相場銘柄の特徴（シグナル）を捉えきれていません。新しい特徴量を追加するか、ラベリング条件の見直しが必要です。

### ② ROC Curve (ROC曲線)
- **グラフの概要**:
  モデルが予測の境界線（閾値）を変化させたときの、**真陽性率（大相場を正しく検出できた割合）**と**偽陽性率（大相場でないのに誤検知した割合）**のトレードオフを示す曲線です。
  - **黒い破線（対角線）**: ランダム予測の基準線（AUC = 0.5）。
  - **AUC (Area Under the Curve)**: 曲線の下側面積。0.5（ランダム）から1.0（完璧）の間で表されます。
- **解釈のポイント**:
  - 曲線が**左上の角に近づくほど（AUCが1.0に近いほど）高い予測精度**を持つ優秀なモデルです。
  - **AUC >= 0.7**: 予測モデルとして妥当な分類能力を有しています。
  - **AUCが0.5付近**: 予測能力がありません（ランダム当てと同じ）。
  - **ロジスティック回帰とランダムフォレストの比較**:
    ランダムフォレストのほうがAUCが大幅に高い場合、特徴量と大相場化の間に「非線形な関係（例: PBRが低く、かつ出来高が急増しているときだけ急騰するなど）」が存在している可能性を示唆します。ただし、訓練データが少なすぎる場合の過学習（過去データへの過剰適合）の懸念もあります。

### ③ Feature Importances / Coefficients (特徴量重要度)
- **グラフの概要**:
  大相場化の予測において、各特徴量がどれくらい貢献したかを可視化した棒グラフです。
  - **左（Logistic Regression）**: 線形的な影響度（偏回帰係数の絶対値）。L1正則化により、予測に不要とされた特徴量は係数が `0`（棒が出ない）になります。
  - **右（Random Forest）**: 分割への寄与度（Feature Importance）。全ての値の合計が `1.0` になります。
- **解釈のポイント**:
  - **バーが長い（上位の）特徴量**:
    大相場発生の強い先行シグナルです。例えば、`volume_ratio_20d` (出来高比率) や `close_dev_20d` (移動平均乖離率) が上位を独占している場合、「ファンダメンタルズよりも直近の需給急増や値動きの勢い（モメンタム）のほうが先行指標として極めて有効である」と判断できます。
  - **両モデル共通で上位にある特徴量**:
    モデルの構造（線形・非線形）を問わず有効な、最も信頼性の高いスクリーニング指標です。
  - **バーが極めて短い、またはゼロの特徴量**:
    大相場銘柄の予測において、ノイズであるとモデルが判断した指標です。スクリーニング条件から除外することを検討する目安になります。

## 5. 運用およびパラメータ調整のヒント

### ⚠️ 積集合（AND）適用時のデータ不足問題
テーマ抽出の条件を積集合（`target_tags.issubset(item_tags)`）に変更したことにより、極めて厳密に関連銘柄が絞り込まれます。
起点の銘柄によっては、**テーマ内の抽出数が数銘柄程度まで激減するケース**があります。

- **データ数が少なすぎる場合の影響**:
  機械学習モデルの訓練データ（レコード数）が絶対的に不足し、モデルの学習が崩壊します。この場合、ROC-AUC は `0.5` 付近に低下し、特徴量重要度も信頼できないランダムな値になってしまいます。

### 🛠️ 銘柄数を適正化するための調整アプローチ
分析を有効に機能させるために、以下の方法で抽出銘柄数を**20〜100銘柄程度**にコントロールすることを推奨します。

1. **タグ定義の緩和（`Sector_Major`のみの適用）**:
   `get_theme_stocks` において、`End_Market`（細かい用途タグ）は積集合の判定から除外し、`Sector_Major`（大枠のテーマ・セクター）のみを積集合の一致条件にすることで、対象数を適度に広げます。
2. **起点銘柄（`TARGET_CODE`）の選定**:
   「半導体」「生成AI」など、市場の関心が高く元々の関連銘柄数が多いテーマに属するコードを選択して検証を行います。
