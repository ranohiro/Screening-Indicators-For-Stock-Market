# 注目テーマ自動検知ロジック検証ノートブック（`sector-detec-param.ipynb`）仕様・解説書

このドキュメントは、日本株投資アプリの新機能「本日の注目銘柄 ＞ 注目テーマ」の実装に向けて、市場の大口資金流入トレンドを検知するロジックを検証・パラメータ調整するために作成されたノートブック [sector-detec-param.ipynb](file:///Users/hiranotakahiro/Projects/銘柄スクリーニング検証/notebooks/sector-detec-param.ipynb) の技術仕様および使用方法をまとめたものです。

---

## 1. 目的と位置づけ
本ノートブックの主な目的は以下の通りです。
- **ロジックの妥当性検証**: 過去の特定相場日（例: MLCC相場が過熱した日など）を対象に、意図したテーマが上位にスクリーニングされるかを確認する。
- **パラメータの調整**: 出来高の急増倍率や高値圏の閾値、時価総額に応じた加重平均や、テーマの「密度（活性化率）」を反映するスコアリング調整の効果を検証する。
- **デバッグとあぶり出し**: 特定テーマ内で条件を満たした銘柄、および条件から漏れてしまった銘柄のパラメータ（出来高倍率・年初来高値からの位置）を可視化し、閾値のチューニングを行う。

---

## 2. 前提データ構造
計算には以下の2つのデータソースを使用します。

1. **テーマ・タグマスター (`ticker_dictionary.json`)**
   - 銘柄コードに紐づくテーマタグ（例: 「セラミックコンデンサー」「半導体製造装置」など）が定義されたマスター辞書。
   - ノートブック内では、銘柄コードからタグを引く `code_to_tags` と、タグから所属する全銘柄コードを引く逆引き用の `tag_to_codes` を構築して利用します。

2. **日足株価データベース (`stock_data.db` 内の `daily_prices` テーブル)**
   - 全上場銘柄の日足データ（日付 `date`, 銘柄コード `code`, 終値 `close`, 出来高 `volume`, 時価総額 `market_cap` など）を格納した SQLite データベース。

---

## 3. 計算アルゴリズムの詳細

### 【Step 1: 動意株のスクリーニング】
毎日、全上場銘柄に対して以下の2つの条件を両方満たす銘柄を「動意銘柄」として抽出します。

1. **出来高モメンタム（急増）条件**:
   - $\text{直近5日平均出来高} \ge \text{過去25日平均出来高} \times \text{閾値（VOL\_RATIO, 初期値 2.0倍）}$
2. **高値圏ブレイクアウト条件**:
   - $\text{当日終値} \ge \text{年初来高値（過去250日高値）} \times \text{閾値（PRICE\_HIGH, 初期値 95\%）}$

### 【Step 2: 個別銘柄のスコア貢献度と外れ値処理】
動意銘柄ごとに、テーマへのスコア貢献度（`stock_score`）を計算します。

$$\text{個別銘柄スコア} = \text{処理済出来高倍率} \times \log_{10}(\text{時価総額})$$

- **出来高倍率の上限クリッピング (`clip_outliers=True`)**:
  極端に出来高が跳ね上がった「モンスター銘柄（仕手株など）」が1銘柄でテーマ全体のスコアをジャック（独占）するのを防ぐため、出来高倍率に上限（`MAX_VOL_RATIO`, 初期値 5.0倍）を設けてクリッピング処理を施します。
- **時価総額ウェイト**:
  時価総額（器）が大きい銘柄ほど、大口資金が流入した際の影響度が大きいため、時価総額の常用対数 $\log_{10}(\text{時価総額})$ をウェイトとして乗算します。
- **ノイズフィルタ（広すぎるタグの除外）**:
  「電気機器」「電子機器」「円安メリット」など、あまりに範囲が広くトレンド特定に適さないタグは、スコア集計から除外するフィルタリング処理が施されています。
  （※本ノートブックでは、`analyze_themes_by_date` 関数内に除外対象のタグが直接ハードコーディングされています。本番実装時は定義ファイルやパラメータからの読み込みへの移行が推奨されます。）

### 【Step 3: テーマスコアリングと密度調整（活性化率）】
テーマごとに所属する動意株の個別銘柄スコアを合算（`計算スコア`）したのち、テーマ全体の連動性を評価するために以下の**密度調整（活性化率）**を適用し、`最終調整スコア`を算出します。

- **活性化率（%）**:
  $$\text{活性化率} = \frac{\text{テーマ内の動意銘柄数}}{\text{テーマに所属する総銘柄数}}$$
- **最終調整スコア（調整ロジック）**:
  $$\text{最終調整スコア} = \text{計算スコア} \times (1 + \log_2(\text{動意銘柄数})) \times (\text{活性化率} + 0.5)$$

> [!NOTE]
> この密度調整により、「1銘柄だけが猛烈に買われているテーマ」よりも、「テーマ内の複数銘柄（あるいは高い割合）が同時に動意づいているテーマ（＝集団連動相場）」がランキング上位に浮上しやすくなります。

## 4. 検証パラメーター設定
ノートブックの「5.検証結果確認」セルで以下のパラメータを変更し、抽出結果の変化を確認することができます。

| パラメータ名 | デフォルト値 | 説明 |
| :--- | :--- | :--- |
| `TARGET_DATE` | `"2026-04-30"` | 検証対象とする相場日（YYYY-MM-DD）。 |
| `VOL_RATIO` | `2.0` | 出来高モメンタムの閾値（倍率）。 |
| `PRICE_HIGH` | `0.95` | ブレイクアウト（年初来高値からの維持率）の閾値。 |
| `CLIP_OUTLIERS` | `True` | 出来高倍率の上限クリッピングを有効にするか。 |
| `MAX_VOL_RATIO` | `5.0` | クリッピングの上限倍率。 |
| `USE_DENSITY` | `True` | 密度調整ロジック（最終調整スコア計算）を適用するか。 |

## 5. 詳細分析・デバッグ機能

ノートブック後半の「6.特定セクターに含まれる銘柄の確認セル」では、閾値設計を微調整するための詳細分析が可能です。

1. **動意銘柄ディテールの確認**:
   - 指定した特定のテーマ（例: `"セラミックコンデンサー"`）において、どの銘柄がどれだけスコアに貢献したかを一覧表示します。
2. **スクリーニング漏れ銘柄のあぶり出し**:
   - 対象テーマに所属しているが、今回のスクリーニングで「動意株」と判定されなかった銘柄を自動抽出します。
   - 漏れた銘柄の「実際の出来高倍率」と「高値からの位置（％）」を表示するため、「出来高は急増しているが高値圏に届いていない（PRICE_HIGHを下げれば救える）」のか、「高値圏にいるが単に出来高が足りない（VOL_RATIOを下げれば救える）」のかといった原因究補とパラメータ微調整に役立ちます。

## 6. アプリケーション（本番）実装へのロードマップ
本ノートブックで検証したアルゴリズムを Streamlit アプリに移植するにあたり、以下の追加実装を行います。

1. **バッチスクリプト化 (計算と蓄積の分離)**:
   - 毎日大引け後に全銘柄のデータを元に本アルゴリズムを実行し、上位ランキング結果（例: Top 30）および牽引銘柄のリストを `daily_theme_scores.json` または SQLite 等の時系列データベースに保存・蓄積する。
2. **前日比・連続流入日数の計算**:
   - `daily_theme_scores.json` の過去履歴と比較し、前日からの順位変動（例：`+5位`, `NEW`）や、Top 20圏内に連続してランクインしている日数（`連続流入日数`）を算出する。
3. **Streamlitでの可視化 (ダッシュボード)**:
   - 算出した注目テーマランキングをテーブル表示し、順位急上昇したテーマ（例: 前日比+5以上）や連続流入3日目のテーマをカラーハイライト等で視覚的に強調する。
   - プレースホルダーとして用意された過去のアーカイブ折れ線グラフ（`st.line_chart`）を利用し、特定テーマのスコア推移を振り返る機能を実装する。
